# 2. 앱플레이어 인스턴스 생성

MumuPlayer를 기준으로 설명하고 있지만, 아래 조건을 만족한다면 어떤 앱플레이어를 사용하더라도 상관 없습니다.

- 해상도 1280x720
- DPI 180
- 프레임 60fps
- adb 디버깅 활성화 (로컬 연결만 있어도 됨)
- Root 권한 활성화

## 2.1. MumuPlayer 설치

<https://www.mumuplayer.com/download/>에서 MumuPlayer를 설치해주세요.

사용하시던 다른 앱플레이어를 쓰셔도 상관없습니다.

## 2.2. MumuPlayer 인스턴스 생성

> [!NOTE]
> 이미 사용중인 앱플레이어가 있다면 알아서 비슷한 설정을 찾아서 사용하세요.

![디바이스 설정 진입](/docs/image/2/mumu_settings.png)

먼저 인스턴스를 만든 뒤 ...을 누르고 디바이스 설정으로 들어가주세요.

![디바이스 설정](/docs/image/2/mumu_settings_detail.png)

아래 설정들은 **반드시 해야 합니다**.

- 보기
  - 해상도 설정: 사용자 정의, 너비 1280, 높이 720, DPI 180
  - 프레임 레이트 설정: 최대 프레임 레이트 60
- 기타
  - Root 권한 활성화: 실행
  - ADB 디버깅: 로컬 연결 활성화

아래 설정들은 사양을 줄여주지만 명일방주 & 매크로가 정상 동작하지 않을 수도 있습니다.  
너무 렉이 걸리거나 명일방주가 이상하게 보이면 적당히 사양을 높여주세요.

- 성능
  - 성능 설정: 사용자 정의, 중앙 처리 장치 2코어, RAM 4 GB
  - GPU 사용 정책: 메모리 사용 감소
  - 스마트 RAM 최적화: 실행
- 보기
  - 다중 탭 모드: 꺼짐
- 소리
  - 시스템 소리 비활성화: 실행

### 2.2.1. 배럭용 인스턴스 생성

배럭을 돌릴 때 **절대** 인스턴스 복제 기능을 사용하면 안 됩니다.  
반드시 모든 인스턴스를 하나하나 새로 만들어야 합니다.

![Mumu 인스턴스 배럭 생성](/docs/image/2/god_mumu_1.png)

![Mumu 인스턴스 배럭 설정](/docs/image/2/god_mumu_2.png)

MumuPlayer는 한 번에 여러 인스턴스를 생성하거나 설정할 수 있으므로 참고하시기 바랍니다.

> [!TIP]
> 인스턴스 이름 뒤에 언더바 숫자를 붙여주면 (ex: XXX_1) 나중에 인스턴스 구분할 때 도움이 정말 많이 됩니다.

## 2.3. 명일방주 설치

PlayStore 대신 [APKPure](https://apkpure.com/%EB%AA%85%EC%9D%BC%EB%B0%A9%EC%A3%BC/com.YoStarKR.Arknights) 같은 곳에서 명일방주 apk를 받아서 설치해주세요.  
**절대** 구글계정으로 로그인하시면 안 됩니다.

MumuPlayer의 경우 apk/xapk 파일을 인스턴스로 드래그하시면 알아서 설치해줍니다.

![명일방주 메인화면](/docs/image/2/fuck_yostar_1.png)

명일방주를 실행시켜 리소스를 전부 다운로드 한 다음, 메인화면에 도달했을 때 START 버튼을 눌러주세요.

![Yostar 연동 상태 확인 화면](/docs/image/2/fuck_yostar_2.png)

그럼 위와 같은 *Yostar 연동 상태 확인* 창이 뜨게 되는데 이 창을 닫아주세요.

배럭을 돌릴 경우 각 인스턴스마다 명일방주 새로 깔고 리소스를 새로 받으셔야 합니다.
